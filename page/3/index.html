财主家的故事

财主家要选一个算账的伙计，于是财主家的门庭开始繁忙起来。来来往往的人都要踩着后脚跟了，财主家的门槛都被磨低了一大截。

这活是个肥差，别看也是要幸幸苦苦加班加点的干。但是能跟掌柜的熟，还掌握着账薄。油水可不少，所以，大家都把头消尖了，打算挤进去。

先是考试，一人发一张白纸，让大家写数，从一写到一万，不能有错。说是选前几个写完的。结果选出来了，大家发现，刘副掌柜的打字不识侄子和钱大掌柜数不清一二的外甥都入选了。顺便还获得了年度最佳书法奖。

不过也没有人发表啥意见，只是钱大掌柜和刘副掌柜家里的灯这几天晚上都亮到很晚。

再接着就是开会，面试。几大掌柜端坐在堂上，高高在上。彷佛庙堂里的几尊佛像，接受人们的朝拜。会议很成功，大家都很满意。最终刘副掌柜的侄子成功加入了啃得起豪华套餐。当然，这位大字不识一个的侄子，入选的理由合情合理。具有创新思维，敢做敢当。

入职的当天，财主家当铺门前热闹非凡，到处都是赞扬的声音。不过这大侄子也倒是敢做敢当，账目记得好坏不知道，别人欠的钱倒是要回来好几笔，只是镇上的人口失踪了几个。

后来，财主家的狗叫了好几天，据说一个穿着长跑的人，跑到财主家门口，满口什么之乎者也。拉着刘掌柜的衣角，问刘掌柜老虎长什么样子？

再后来，财主家的狗也不叫了，来的都是提腊肉的人。但若是门口来了要饭的叫花子，他还是吠得厉害，要是有人来门口之乎者也，他就恨不得把自己得牙齿拔下来，扎住那人的嘴。

之后的事情就不得而知了，财主的院子外，又筑了一道高墙，这下，那些只会之乎者也不拎着东西的人，连财主家的大门都不到了。人们只是注意到，高墙里进进出出的人越来越少，狗也渐渐不叫了。

把头埋在沙子里

以前有种生物叫做鸵鸟，后来灭绝了。

据说鸵鸟有个习惯，一遇到事情，就把头埋到沙子里。其实，一开始鸵鸟没有这个习惯。他旁边也没有沙子。

后来，旁边总是有其他动物找鸵鸟帮着忙，帮那忙。鸵鸟都帮，但帮完后，大家又说打扰鸵鸟，不应该找鸵鸟的。有一次，鸵鸟在把东西丢在沙滩上了。他在沙子里找的时候，有动物又来找他。他没听见，后来，鸵鸟喜欢把头埋在沙子里的说法就传扬开了。

至于鸵鸟是怎么灭绝的。那是因为，鸵鸟实在想证明自己的头不在沙子里，于是，就打算把头拿下来，挂在高高的树上。但他忘了，头拿下来，就活不了了。于是他长长的脖子就倒在沙地上，大家就更加确定，鸵鸟的头是埋在沙子里的了。

好的产品·软件类

2020-11-29

我最近两年内一直在重度使用手机，电脑等电子产品。由于经济等因素，主要的体验都是在软件类别上，通过对上千个软件的尝试，觉得抛开经济，能力等因素，单纯的就一个软件而言，怎样才能算一个更好的产品？

好的产品，要有两个面孔。对于普通大多数不喜欢折腾的用户和比较喜欢折腾的用户都提供优秀的体验

**普通用户 :

这些用户是产品的绝大多数的使用者，他们需要一个体验很好的软件，满足自己的主要功能。而产品的盈利也主要来源于给他们提供好的服务。所以，一个产品在第一界面和功能上应该是面向此类用户的。

折腾用户：

此类用户喜欢折腾与探究，使得产品满足自己更多的功能和需求，这类用户可以共同维护，产生更好的产品功能与体验，所以，产品的设计应该提供他们创造更多可能的途径。**

软件定位

软件的定位一定要清晰，本质上应该是要解决一类问题，那么做好核心功能是重中之重，但是，在核心功能外的延拓也是必不可少的。所以，首先要有一个清晰的定位，开发一个产品的目的是为了解决什么问题，同时关注问题的发展与变换。及时的跟进与跟新。这是做好产品要思考的第一个问题。

比如，一款输入法的首要目的是进行输入，那么首先要考虑清楚的就是目标范围，要做的就是中文输入还是日语输入。这个问题一般看上去似乎很简单，但实际上，清晰的定位并不是个简单的事情。只有定位清楚目标，才能更好的分清楚功能，交互等安排。

核心功能

知道软件定位后，一定要弄清楚的第二个问题是——核心功能。噱头，营销，可能会取得良好的收益等其他目的，但得不到真正好的产品。但是要知道什么是核心功能，第一件事就是定位要明确。

以中文输入法为例，核心功能就是输入，中文的输入。那么产品的首要目的就是使得用户能够更快、更方便、更准确的输入。那要做的事情就是优化输入联想功能，可以从大数据中获得常用词，或者建立个人的词库。大数据的词库，对于用户第一次使用体验有着极大的影响，决定了用户是否想继续使用这款产品，而用户个人的数据库，则能够使得用户对产品的依赖。随着用户使用时间的增长，输入准确性提高。这样，用户如果切换到其他同类产品，就能感受到本产品在使用准确性的优势。

界面和交互

界面

界面是一个产品体验的关键因素，据不科学研究表明，一个人见面不到1秒内就会对一个人产生感觉的分类，喜欢或者厌恶。好的颜值不论在人或者产品身上都是至关重要的，如果很丑，可能你都没有机会展示自己的内在，就会错过。所以，界面和交互，是产品吸引力的重要因素。所以，首先要提供给用户一个或者多个符合多数人或者不同目标群体的界面。

以输入法为例，在默认皮肤上应该给很多人眼前一亮的感觉，提供几套不同类型好看的皮肤，比如，简约，卡通，炫酷。在使用前让用户选择自己的喜好。

同时，给界面更多的可能性，相比其他输入法，百度皮肤的可能性就要优秀很多，这样能够给产品的界面更多可能。对于折腾用户提供了一个创造的接口。同时应该提供一个良好的创造平台，让更多人能从创造中获利。在这一点上，百度输入法并没有做到，甚至都没有一个好的论坛。主要都在在酷安等其他途径获得皮肤的创造，而且，百度输入法虽然提供了皮肤创造的可能，却连工具都没有好好做，甚至没有说明。

交互

交互也是重要的一个环节，那些功能比较常用，如何更方便的操作，这是很重要的一点，很多产品具有很多好的功能，但在交互上设计的不可描述，导致体验极差。如我文初所说，提供给大多数用户一个简单的初始界面，同时还应提供更多的可定制性功能。

交互的设计首先就是要考量用户的使用习惯，输入法中，很多地方其实都是一样的，但比如输入键的位置，标点符号的位置，在输入过程中会影响用户的体验，同样应该多设计几套初始方案，并给用户简单可以调整的可能。

更多功能

产品会随着时间变换，有相应新的需求与改动，那么新的功能和改动应该随着而来，并且实时关注不同的用户体验和反馈，吸收同类产品好的地方，这样才能使得产品朝着一个良好的方向发展。

例子：

以输入法为例，假如我要做一个输入法，我有那些需要改进的地方。

现在我主要使用是是百度输入法，同时试过一些其他输入法，在此提出一些自己看法。

首先，几个大厂的输入法在核心输入功能上依靠大数据做的还是相当可以的。这一点上是特别好的，支持九键，24键，14键等方面都比较普遍。在九键输入上对准确性的优化是需要注意的一点。

界面上，百度的输入法，可定制性确实很高，但是百度界面开发软件的更新和维护几乎是零。对于社区的维护也是没有的。可以优化一个好一点的界面开发软件，让大多数人都有能力去设计自己的皮肤，其次可以设置一个比较好的商城和社区，让开发者有正规的渠道沟通交流。

在交互上，可以添加界面手势功能，滑动输入功能，提供更好的交互体验。

其他功能上，bd输入法做了很多好的尝试，但都没有做好。

  1. 通知栏剪贴板功能的体验性还是相当可以，但是既然剪贴板可以存放大量信息，为什么不设计搜索功能呢（搜索剪贴板内容）？百度同时设计了一个直接搜索的功能，其实这个功能更好的是调用系统的浏览器打开，或者自己设计一个与浏览器互动的功能。

  2. 同时设计了便签功能，但这个便签使用真的是一言难尽。悬浮窗打开确实可以，同时可以考虑从图标打开，并且，便签功能并未做好，稍微改进一下，做一些分类功能会更好，桌面小插件。

  3. 语音速记功能可以和便签进行整合，进入语音速记的方式真的很难找到。

  4. 同步能力，应该是因为放弃pc端了，所以，pc端的百度输入法完全没有更新，而QQ输入法的剪贴板同步功能在百度输入法中完全没有。

  5. 翻译，ocr，表情等功能可以直接通过输入模式进入，比如，要翻译为英语，可以直接输入翻译 要翻译的文字 实现。

产品·手机浏览器

2020-11-29

早上看到一篇文章，然后为了更好的阅读体验和能过够保存文章，我就又花了一个下午的时间去尝试了各种浏览器，然后，将自己关于手机浏览器的一些想法记录下来。

核心功能

打开网页是浏览器的核心功能，所以，做好打开网页功能是首要的任务，打开速度和加载的形式都是很重要的功能。

界面交互

首先界面的好看性和交互性很重要，而颜值来说，支持自定义是很重要的一方面。而浏览器给人的第一颜值就在于主页的设计。其次是功能区的设计，这两方面，夸克和灯塔浏览器的设计我比较喜欢。

其次，适配全面屏和去掉状态栏和输入栏也很影响使用的体验。能够提供更大的视野与更简单的界面始终是件让人舒服的事情，神奇浏览器的界面的适配在我使用中比较舒服。

动画的加入可以使得过度更加舒畅，让人有更优良的体验。

交互方面，由于现在手机都偏大，一个手无法操作整个界面，所以，首页的输入框在中下的位置会比较舒服，而功能栏在下侧或者可以放在侧面（可能会与手势冲突）。当然提供一个自定义的方式是更好的选择。kiwi，谷歌等浏览器上方的标签页在大屏上体验并不是很好。

主要的一下功能放在主要菜单选项里，其余的可以放在其他位置，或者二级菜单。（我比较支持其他位置，比起二级菜单，少操作一次的感觉还是很好的），可以参考神奇浏览器右上角的下拉功能或者夸克的二级菜单。

手势的支持会带来不一样的体验，左右滑动前进后退，但要注意位置和体验感。

附件功能

edge和chrome在pc端的优良表现，使得手机选择这两款应用有更好的同步体验，但这两款软件手机端的体验并不很好。

可以试着支持chrome的插件，阅读类插件在手机端体验并不好，优化插件展示的形式。

广告拦截，是提升用户体验的一大利器，如何更好的拦截广告是值得注意的。

阅读模式，在众多阅读模式中，我最终还是觉得神奇浏览器的阅读模式比较好，但可惜的是，我想要将这种舒服的排版导出到pdf或者图片，却没有找到这个功能。阅读模式的体验主要就是几点：

  • 字体（大小、样式）
  • 段落
  • 翻页的方式
  • 背景
  • 书架模式等其他

视频观看是很多用户的需求，我不怎么喜欢手机看视频，但确实是很大一块需求，在这一点上，雨见浏览器强大的能力值得参考，对默认视频播放工具的改进是必须的。

下载管理和视频缓存功能中应该参考管理类软件，让用户有更良好的体验。

其他专业类的功能，比如调试，开发等，提供相应的接口。能够使得软件变得更好。

夸克的ocr功能确实不错，但我觉得这不应该是浏览器该做的的事情，而应该是办公学习类软件的事情。

总结：

总的来说，是很难找到一款让我满意的软件，现在主要使用，夸克做日常的搜索和下载，edge偶尔同步页面。而神奇浏览器好看的阅读界面让我在阅读网站文章时会想到它。

分类方法的分歧

如果您有过使用 WordPress 的经验，就很容易误解 Hexo 的分类方式。WordPress 支持对一篇文章设置多个分类，而且这些分类可以是同级的，也可以是父子分类。但是 Hexo 不支持指定多个同级分类。下面的指定方法：

categories:
  - Diary
  - Life

会使分类Life成为Diary的子分类，而不是并列分类。因此，有必要为您的文章选择尽可能准确的分类。

如果你需要为文章添加多个分类，可以尝试以下 list 中的方法。

categories:
- [Diary, PlayStation]
- [Diary, Games]
- [Life]

此时这篇文章同时包括三个分类： PlayStation 和 Games 分别都是父分类 Diary 的子分类，同时 Life 是一个没有子分类的分类。

excerpt 说明

  • 在文章中使用< !–more–> 手动进行截断
    这种方法可以根据文章的内容，自己在合适的位置添加 < !–more–> 标签，使用灵活，也是Hexo推荐的方法。
  • 在文章中的front-matter中添加description，并提供文章摘录
    这种方式只会在首页列表中显示文章的摘要内容，进入文章详情后不会再显示。
  • 自动形成摘要，在主题配置文件中添加
    默认截取的长度为 150 字符，可以根据需要自行设定
auto_excerpt:
  enable: true
  length: 150

建议使用 < !– more –>（即第一种方式），除了可以精确控制需要显示的摘录内容以外， 这种方式也可以让 Hexo 中的插件更好的识别。

随着嫦娥100号离开地球，飞往月球。指挥室里的所有人都屏住呼吸，死死的盯着屏幕上那传回来的实时影像。

好，这里又有个很有趣的论点。分解工作会制造麻烦，如果一个人都有能力去完成所有工作，他肯定知道工作该如何分解。那么是不是就是说，干活的能力比领导的能力强，领导都不会分解工作，而员工可以分解好工作而且自己独立完成。

而且不是只是骨干员工加班，是几乎所有人都加班。

这个方法是好的方法，但用互联网公司历史不长作为说辞就有点勉强了。两个人磨合一年都可以结婚，你是什么工作需要这么长的磨合。一般工作磨合一两个月都可以配合很好了吧，况且互联网从业者都是较高智商的人群，学习能力应该不弱。

况且，公司如果有自己完善的规则，如何规定接口，制定一个大家遵守的标准。对接是很简单的事情，是不是可以说，公司内部管理混乱，无章可循，所以导致员工对接不顺。

“自由”哈哈，如果一个公司能够给年轻人公正的待遇和晋升途径，我猜，没有几个年轻人愿意要这种自由吧。很显然，年轻人之所以频繁跳槽的原因，是因为，给公司付出了自己的青春和劳动后，却得不到应有的回报，和看不到前进的路途吧。

如果公司正真器重年轻人，培养年轻人，给年轻人发展的道路，我想没有几个人愿意跳来跳去，你以为是玩跳一跳吗？

还不都是，将年轻人身上的价值榨干，然后丢弃他们，美其名曰，给社会输送人才，给年轻人发展的自由。

不是，我们要做的，就是要联合起来，让这些剥削我们的人看到力量，他们才会畏惧，才会退让。我们要做的，就是让国家和政府听到我们的声音，党和政府就会去限制他们。

这句话也没问题，但一定注意，要谈效率，但不是只谈效率，否则就是掉进一点概全的错误中去了。

竞争是必然存在的，但国家和政策要保证竞争是朝着良性发展的方向。而不是放任其不管。

现在问题不是一个人加班，而是普遍存在超额加班的情况，一个人加班，就罚一个亿当然是过往矫正。但如果公司有百分之三十，五十甚至更过的员工每天都工作十二个小时。那么不罚几十亿便是对劳动人民的不负责任。

脱离效率，谈制度是不可的，同样只谈效率也是错误的。你能够产生先进生产力的制度，总是会打败其他制度。

为什么复杂的项目会推给国外，我想想他想说的论据是说，

先进生产力总是站在最广大人民群众这一边。

无产阶级还对自己不够狠吗？把生命都献给了资本，还要什么，还要的是我们的灵魂，还要让我们相信，被剥削是你的幸运，996是福报。果然够狠。

我们是要看历史的，虽然人类生产力的发展确实有很大进步，但人性和社会的问题并灭有巨大的突变。而很多人已经探讨社会的发展。很显然，历史上有一条清晰的道路。

不谈主要矛盾，剥削的问题，而谈次要的问题，效率问题。这是主次不分，不抓主要矛盾，而夸大次要矛盾。

现在最有效的路径就是在现有制度下与改良，去监督。不然，放任资本去剥削？

那么这件事情到底应该怎么办，我觉得政府应该去管理和限制这种情况。

我觉得国家应该完善相应法律制度，严惩相关企业。这样，刚开始的时候，可能会因此而降低员工工资，低工资使得不会有那么人再愿意涌入这个行业。从而公司缺少人才，不得已放弃一部分剥削，提高工资，吸引人才，这样才是一个正向发展的过程。

而更多人才流入其他行业，同时可以促进其他行业的发展。

我们的制度是不够完善，这点我们不必掩盖，但我相信党和政府会不断努力，创造一个更加公平和谐的社会。

在矛盾中发现问题，解决问题，这是一个螺旋前进的过程。这才是马列主义，辩证前进的过程。

最后，我想跟大家说的是，大家在看任何言论视频的时候，一定要有自己的思想和判断，要用辩证唯物的方式去判断。当然，我的一些观点也不一定完全正确，大家可以讨论，可以商讨，可以辩论，只有这样才能得出正确的结论，真理总是从群众来，到群众中去的嘛！
